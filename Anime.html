ANIME RECOMMENDATION USING MYSQL¶

A. ABSTRACT¶

Anime is a form of hand-drawn or computer-generated animation originating from Japan. It used to be mistaken as cartoon or a child-like form of entertainment but has recently gained popularity during the Coronavirus lockdowns of 2020. The prohibition of mobility has forced people to seek recreation online, and when Netflix can no longer accommodate their tastes, that is when anime shines. And it wasn't just the visual, but studios have also invested in great soundtracks and most importantly, the plots.

Contrary to popular beliefs, anime not only provides its fan with a wide variety of genres, but also complex plots and character developments that were adapted for adults. The most common example being Shingeki di Kyojin (Attack on Titan).

Which raise the question of how could these new-comers experiences be improved and in which will be the main objective of this study.

The dataset that our study will be based on is Anime Recommendation Database 2020 available on Kaggle

This dataset contains information about 17.562 anime and the preference from 325.772 different users on https://myanimelist.net/.

B. DATASET¶

I.TABLE DESCRIPTION¶

The database contains 4 tables saving information on the website as mentioned above. They all have missing data which is marked as Unknown because it is up for interpretation depending on the attribute. It could mean Missing Value, Others, or Not Available.

The first table "anime.csv", included all pieces of information related to the animes shown on the site.

Each anime has a unique code MAL_ID, Name
Score as an average of all user ratings
Genres, an anime can be classified into a list several genres
The English name, Japanese Name
Type, animes with the same name can be presented using a different type.
Aired, Premiered time written in text format
Producer, Licensors, Studios, an anime can be produced by several at a time, and different seasons can be produced by different studios
Source of content
Duration, Rating in text format
Ranked based on the score
Popularity, Members, Favorites, Watching, Completed, On-Hold, Dropped, Plan to watch - User interaction with animes
Score 10 - 1: numbers of users who gave different scores.

The table anime_with _synopsis.csv contains anime MAL_ID, Name, Score, Genres similar to anime.csv, and synopsis as the description of the anime.

animelist.csv contain user_id, anime_id, rating, watching status, watched_episodes. And user can rate or watch unlimited anime, watching status are identified using id

rating_complete contains user_id, anime, and rating of anime give the user has completed watching it.

watching_status includes ID of the status and status description.

So it might be worthwhile to investigate the reason and hopefully find a better method for evaluation.

From above, it can be inferred that there are 2 main reasons for missing Score:

  1. Premiered time:

Many of these animes were released very much long ago (1961 - 2000), which make them unpopular among current users or doesn't accomodate to the current taste. Another being it hasn't been realeased yet.

However there are also very sought after anime that hasn't been released but already received ratings, this is due to them being based on very famous manga and very successful past seasons. It also contrast our observations on number of user who considered it their favourite.

  1. Not Favourite:

Something that all these animes has in common is how little users like them. Reminded that, favourite shows the number of users who considers the anime their favourite, the average being 458 users. However for those animes without score, most of which have zero to none user who likes it.

Hypothetically, Score is our best method in evaluating how good or bad or well-perceived and anime is. Because it takes into account all the user that have watched it and have given score to it. But, is there an alternative?

3.5 Anime Genres of the TOP 50

Anime Genres is saved as Text Datatype in our database and is presented as a list of genres separated by a comma.

Since MySQL doesn't support the command UNNEST, we must once a gain use the help of Python to unpack our lists, and count the number of occurence of each genre.

Note that the sum of genre occurences will not be equal to the number of anime we have, since each anime consists of several genres, so the data is repetitive.

In [13]:
def Count(colname,tablename):
  input = tablename.loc[:,colname]
  output = []
  for row in input:
    out  = row.split(", ")
    for word in out:
      output.append(word)
  counter = Counter(output)
  i = 0
  for val,count in counter.most_common():
    print(val,count)
    i += 1
  print("The number of genres is:{}".format(i))
